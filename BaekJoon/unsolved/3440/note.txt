# 
오래 걸릴 것 같은 문제, 계속 풀기


1. 그림 해석 
- 그림을 기반으로 그래프로 변환,
    + "#" : 게이트를 의미, 직사각형, 각 게이트는 붙어있지 않음
        - 왼쪽 변은 input, 오른쪽은 output
        - output은 Negate 될 수 있다
        - 한 변은 최소 3의 공간을 차지
        - 내부는 하나 빼고 빈 공간, 
        - 그 하나는 "1", "&", "=" 중에 하나
    + "-" : 수평 라인, 무조건 연결
    + "|" : 수직 라인, 무조건 연결
    + "+" : 구부러진연결, 연결안된 선이 근처에 있을 수 있으니 조심해야함
    + "x" : 두 라인이 교차, 왼쪽은 오른쪽, 위는 아래랑
    + "=" : input 또는 output을 의미
        - 왼쪽에는 value source
        - 오른쪽은 value consumer
    + "o" : Negation을 의미, 
    + "0"/"1" : circuit의 입력, 항상 오른쪽의 = 와 연결됨
    + "A" - "Z" : 각 문자는 최대 한번만 등장하고, 왼쪽의 =와 연결됨
    + "*" : 마지막 입력 of TC
한 행은 최대 200 문자임

-  보드 서치해서 
    // -> 숫자가 나오면 노드를 만들고 따라감
    //   -> # 이 나오면 노드를 만들고 연결시킴
    //  위 과정을 DFS 마냥 수행함.
    #을 먼저 파악해서 노드를 만들고, 우선 입력과 output의 = 위치를 기록
    -> 반례 1=A

    숫자 기준 서칭이 맞는 것 같고, 
    노드는 좌상단을 기준으로 identify
    DFS 방식으로 하자
    숫자도 노드, 게이트도 노드
    위상정렬로 수행하고, 수행하면서 간선을 끊는것 + 결과값을 넣어줘야함.
