# 
묶었을 때 최대?
어떤 수를 묶던 상관이 없다..


음수는, 음수끼리 묶던가 안묶던가
양수는 큰수끼리 묶어야함
0은, 가장 큰 음수랑 묶던가 안묶던가

음..
크게 만드는 요소인지,
작게 만드는 걸 막는 요소인지 구분 필요
(크게 만드는 요소가 우선순위 높음)
말이 애매한데... 그냥 직관적으로...

1. 묶을지 말지
2. 어떤 걸 묶을 지

1. 묶을지 말지
    음수 - 음수 : 묶기 (크게 만드는 요소)
         - 양수 : 안 묶기 (작게 만드는 걸 막는 요소)
         - 0 : 묶기 (작게 만드는 걸 막는 요소)

    양수 - 음수 : 안묶기 (작게 만든느 걸 막는 요소)
         - 양수 : 묶기 (크게 만드는 요소)
         - 0 : 안묶기 (크게 만드는 요소)


2. 어떤 걸 묶을 지
    절댓값 기준, 큰 요소끼리 묶기

3. 고려할 케이스
    -> 음수는 생각해봤는데, 양수는 생각 못했었음
    예제처럼 1 1 이 들어오면 안합치는게 맞음

그냥 
음수끼리 minHeap
양수끼리 maxHeap

남은거 있으면
음수, 양수, 0 이렇게 세 요소가 0개 또는 1개나 나올텐데 그떄 알아서 생각하면 될듯?
